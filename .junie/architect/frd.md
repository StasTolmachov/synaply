Functional Requirements Document (FRD): MVP Платформы Synaply
Статус: Ready for Development
Цель: Фиксация функционального скоупа минимально жизнеспособного продукта (MVP) для быстрого запуска пилотов в норвежских коммунах (Voksenopplæring). Описание базовой бизнес-логики, архитектурных ограничений и комплаенс-требований.
Концептуальная архитектура MVP
Synaply MVP — это сфокусированная, облегченная версия платформы. Главная цель этапа — обеспечить максимально быстрый Time-to-Market с минимальными затратами ресурсов, доказав ключевую ценность продукта: автоматизацию физиологического процесса запоминания слов и высвобождение времени учителя. Архитектура намеренно упрощена для защиты от over-engineering, но закладывает надежный фундамент для будущего масштабирования.
Модуль 1: Student App (Интерфейс ученика)
Функционал ученика сфокусирован на атомарном обучении и преодолении языкового барьера без отвлечения на сложные социальные механики.
2.1. Идентификация и доступ
Feide SSO (OIDC): Бесшовная авторизация через национальную норвежскую систему. Является абсолютным блокером (Must-Have) для выхода на B2G-рынок Норвегии и проведения пилотов в школах.
2.2. Core Learning Loop (Основной цикл обучения)
Интервальное повторение (FSRS v4): Математическое ядро, вычисляющее оптимальный день следующего повторения карточки.
Механика тренировки: Исключительно текстовый ввод с клавиатуры (формирование мышечной памяти).
Diff-анализ: Мгновенная проверка ответа с подсвечиванием опечаток (красным/зеленым).
Озвучка (TTS): Использование встроенного в браузер Web Speech API для снижения операционных расходов (COGS) на этапе MVP.
2.3. Инклюзивность и ИИ-Тьютор
AI Tutor in Your Pocket: Интеграция с API Gemini 1.5 Flash. При совершении ошибки ИИ динамически генерирует объяснение грамматического правила или контекста на родном языке ученика (поддержка 114+ языков).
2.4. Визуализация прогресса
Базовое Дерево Навыков (Skill Tree): Отказ от токсичных таблиц лидеров (Leaderboards) в пользу личной визуализации. Ученик видит, как "растет" его мозг (нейронная сеть) по выбранной профессии. Обеспечивает критически важный Aha-Moment для конверсии 7-го дня.
Модуль 2: Teacher Dashboard (Интерфейс учителя)
Панель преподавателя в MVP решает исключительно проблему нехватки времени на подготовку к урокам и администрирование.
3.1. Генерация контента и учебные планы
1-Click AI Generation: Инструмент для мгновенного превращения любого скопированного текста (новости, инструкции) в структурированный глоссарий для изучения.
Базовые Career Paths: Наличие 3-5 предзаполненных профессиональных треков (например, "Строительство", "Клининг", "Уход за больными") "из коробки". Позволяет учителю запустить пилотный урок в первый же день без необходимости самостоятельной генерации материалов.
3.2. Управление и базовая аналитика
Синхронизация классов: Интеграция с Feide Groups API для автоматического подтягивания списков учеников.
Статистика выполнения: Упрощенный дашборд, отображающий базовый процент выполнения назначенных списков слов учениками.
Отложенный функционал (V2 Scope)
Для обеспечения запуска в течение 2-3 месяцев, следующий функционал жестко исключается из Scope MVP:
Live Engagement Radar: Требует сложной WebSockets/SSE инфраструктуры.
P2P Roleplay & Live AI Chat: Отложено из-за непредсказуемых расходов на API и необходимости сложной модерации (guardrails).
Кооперативная геймификация (Dugnad): В MVP остаются только личные "ударные режимы" (streaks).
Speech-to-Text (Голосовой ввод): Отложено из-за сложности обработки акцентов.
ClickHouse (OLAP Телеметрия): В MVP вся базовая аналитика пишется и читается напрямую из PostgreSQL.
Технические и комплаенс требования (Блокеры для продаж)
MVP проектируется с учетом жесточайших требований муниципального B2G-сектора.
5.1. Архитектурный стек
Frontend: Next.js (React) + TailwindCSS, реализованный как Progressive Web App (PWA) для обхода 30% комиссий App Store / Google Play.
Backend: Golang. Паттерн Modular Monolith со строгой изоляцией доменов в коде.
Database & Cache: PostgreSQL (RDS) с настроенной изоляцией данных арендаторов (Tenancy/RLS) и Redis для фоновых задач и Rate Limiting (защита от Thundering Herd).
5.2. AI Routing и Маршрутизация затрат
Простые переводы направляются через DeepL API (низкий COGS).
Анализ ошибок и генерация контента маршрутизируются в Gemini 1.5 Flash.
5.3. GDPR & Privacy by Design
Zero Data Retention: Архитектурная гарантия автоматического удаления логов транзакционных чатов с ИИ.
Data Residency: Использование серверов для БД и API исключительно в зоне EU/EØS (Франкфурт/Стокгольм). Без этих двух пунктов подписание Databehandleravtale (DPA) с коммунами невозможно.
